## 🎯 运行框架特点

### 1. 双Agent架构

```
主Agent (协调者)          子Agent (执行者)
    ↓                          ↓
• 模式判定                  • 代码审查
• 预扫描编排                • 问题发现
• 交互式确认(纯文本)        • 报告生成
• 快速启动参数校验          • 飞书上传
• 参数准备与注入            • 结果汇总
• 结果展示
```

**优势**：
- 主Agent专注用户交互和流程控制
- 子Agent专注代码审查任务
- 长时间审查不阻塞主Agent
- 用户可在审查期间继续使用OpenClaw

### 2. 双模式运行

| 维度 | 交互式模式 | 快速启动模式 |
|------|-----------|-------------|
| 触发 | 不含 `--mode` | 包含 `--mode` 参数 |
| 适用场景 | 人工审查 | 定时任务/CI/CD |
| 用户交互 | 逐步确认（6步） | 无（跳过交互） |
| 参数来源 | 文本选项交互 | `--` 参数直接传入 |
| 参数校验失败 | 提示重试 | 直接报错终止（不降级） |
| 执行计划确认 | 必须 | 跳过 |

### 3. 严格单步交互规则

交互式模式下，主Agent必须遵守以下约束：

| 规则 | 说明 |
|------|------|
| 工具禁用 | 禁止使用 `AskUserQuestion` 等结构化输入工具 |
| 纯文本交互 | 所有选项以普通文本输出，等待用户自由输入 |
| 单步执行 | 每个回复只能包含一组交互选项 |
| 禁止合并 | 不得将多个步骤（如审查范围+审查模式）放在同一回复 |
| 禁止跳过 | 即使参数看似明确，也必须逐个展示选项 |

### 4. 脚本化工作流

**优势**：
- 脚本可独立维护和测试
- 修改脚本无需更新文档
- 输出格式标准化便于解析
- 可复用于其他工具

### 5. 引用式架构

## 🚀 执行流程总结

### 交互式模式

1. **用户触发** → 主Agent接收请求
2. **预扫描** → 4脚本顺序执行采集项目数据 → 输出预扫描摘要
3. **交互式确认** → 6步纯文本选项交互（严格单步）
4. **阶段六** → 构造prompt并调用子Agent
5. **子Agent执行** → 读取审查框架 + 扫描代码 + 生成报告
6. **飞书上传** → 条件执行（根据用户选择）
7. **结果展示** → 主Agent展示结果给用户

### 快速启动模式

1. **用户触发**（含 `--mode` 参数） → 主Agent接收请求
2. **预扫描** → 4脚本顺序执行采集项目数据
3. **参数校验** → 校验必填参数，失败则终止（不降级为交互式）
4. **阶段六** → 直接构造prompt并调用子Agent（跳过交互确认）
5. **子Agent执行** → 同上
6. **结果展示** → 主Agent展示结果给用户

## 🆕 版本 5.2 更新说明

### 预扫描阶段重构
将原来的阶段一至阶段四合并为统一的「预扫描阶段」，4个脚本按顺序自动执行，不与用户交互，全部完成后统一输出预扫描摘要。

### 快速启动模式
新增快速启动模式，支持定时任务、自动化脚本、CI/CD 集成等无需人工交互的场景：
- 通过 `--mode` 参数触发，跳过所有交互式确认
- 完整的参数规范（`--mode`、`--type`、`--scope`、`--branch`、`--upload`）
- 严格的参数校验，缺失必填参数直接报错终止，不降级为交互式模式

### 强化交互式确认规则
针对模型合并多步骤选项的问题，新增以下规则：
- **工具禁用**：禁止使用 AskUserQuestion 等结构化用户输入收集工具
- **严格单步执行**：每个回复只能包含一组交互选项
- **典型错误反例**：明确展示"审查范围+审查模式"合并展示的错误场景
- **步骤边界约束**：每个步骤结尾明确标注"不得同时输出下一步骤的选项"

### 多维表格字段扩展（15 → 18）（v5.1）
新增 3 个预留修复字段，用于完整的问题修复追踪：
- **修复时间**（日期）：记录实际修复完成时间
- **修复分支**（文本）：记录修复所在的分支名（如 fix/issue-123）
- **修复人**（人员）：记录实际修复人员

这些字段初始留空，供后续修复流程更新使用。

### 移除 Python 依赖（v5.1）
- 将时间戳获取从 `python3 -c "import datetime..."` 替换为纯 Bash 命令 `date +%s%3N`
- 所有脚本保持纯 Bash 实现（`#!/bin/bash`）
- 确保在仅支持 Bash/Shell 的标准环境中正常运行
